# unzip-dongpo · 解压苏东坡

> 输入一个名字，能展开成一个世界。
> AI 时代，每个人的名字都是一个 prompt。

一次实验：把一个历史人物从大模型里**结构化地解压**出来，而不是只问一个问题得一个答案。

![screenshot](./screenshot.png)

## 怎么玩

双击 `index.html` 打开。自包含单文件，无构建无依赖，直接跑。

- 顶部横向时间轴切换 **8 个人生切片**：眉山 → 汴京 → 密州 → 黄州 → 杭州 → 惠州 → 儋州 → 常州
- 每个切片 = **引文 · 散文 · 身份 pill · 六层维度 · 维度交织点 · 钻取按钮**
- 散文和分层卡里的人名、作品、专有术语都是**可点的钩子**（棕红/info 双色分层）
- 右侧对话面板：填入 Anthropic API Key，和**此刻这个时期的苏东坡**对话
- 切换切片时对话自动封存，system prompt 重新锚定到新的时空
- 手机端：手势左右滑动切换时期，进度点指示当前位置

## 核心想法

一个人不是几个平行的维度列表，而是所有维度在**某一时刻**交织在一起的活体。

黄州的那个冬夜，他同时是：
一个被贬的罪臣、一个饥饿的农夫、一个和朋友喝酒的中年人、一个想弟弟的哥哥、一个刚发明了新菜的厨子、一个书法家、一个佛学徒、一个父亲、一个思想者。

分成五个抽屉看，人就死了。缝合回一个瞬间，人才活。

所以本项目把苏东坡解压成 **8 个可居住的时空瞬间**，每个瞬间有：

- **散文**（把所有维度缝回一个活人，约 250 字）
- **六层结构**（身份、身边的人、手上、桌上、精神状态、远方）
- **维度交织点**（明确写出这一刻的张力）
- **若干钩子** 通往对话

## 对话苏东坡

对话直连 Anthropic Claude，API Key 仅存本地 `localStorage`，不上传任何服务器。

**关键设计**：每次发送时 system prompt 动态注入当前切片作为**时空锚点**——所以你问的不是抽象的苏东坡，而是 1082 冬夜黄州的他，或 1097 儋州桄榔庵里的他。切片一变，对话自动封存，锚点重设。

## 为什么是苏东坡

不是因为他"伟大"，而是因为他留下了足够密的**互相印证的锚点**——3000 首诗、几十篇文、书法、画、同时代人的日记。

**锚点密度 = 模型中的分辨率**。

推论：未来每个普通人的"解压度"，取决于他在数字世界留下多少锚点。这个工具：
- 第一批服务历史名人
- 第二批服务任何愿意给自己喂数据的人

## 技术栈

- 纯 HTML/CSS/JS 单文件，无构建无依赖
- Noto Serif/Sans SC（Google Fonts）
- 宣纸色调 + 锈红色系
- Anthropic Claude Sonnet via 浏览器直连（`anthropic-dangerous-direct-browser-access`）
- 手势识别：touchstart/touchend swipe
- 响应式：480px/600px 两级断点
